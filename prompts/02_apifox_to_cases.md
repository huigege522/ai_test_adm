# 阶段二-A：Apifox 接口文档 → pytest 测试用例生成

**使用场景**：从 Apifox 获取接口定义后，让 AI 批量生成覆盖全面的 pytest 用例。  
**在 Cursor 中使用**：通过 Apifox MCP 直接读取，或粘贴接口 JSON 定义。

---

## 方式一：通过 Apifox MCP 直接读取（推荐）

```
使用 Apifox MCP 读取项目 [项目名称] 中 [模块名] 模块的所有接口。

针对每个接口，生成完整的 pytest 测试类，要求：

1. 【正常场景】200 状态码 + 业务码为 0 + 必填字段存在性断言
2. 【参数缺失场景】逐一缺少每个必填参数，期望返回 400/422
3. 【参数类型错误】字符串传数字、数字传字符串等
4. 【边界值场景】字符串字段：空字符串、最大长度+1；数字字段：最小值-1、最大值+1
5. 【越权场景】无 Token 返回 401，低权限 Token 返回 403

代码规范：
- 使用 tests/utils/api_client.py 中的 ApiClient 类
- 使用 @pytest.mark.parametrize 合并同类边界值用例
- 每个 class 对应一个接口，命名如 TestUserCreate
- 添加中文描述的函数名，如 def test_正常创建用户(self)
- 在文件顶部注明接口路径和 Apifox 接口 ID

输出为完整可运行的 Python 文件内容。
```

## 方式二：粘贴接口定义（Apifox 导出的 JSON 或手工描述）

```
以下是接口定义，请生成完整的 pytest 测试用例：

接口名称：[接口名]
请求方法：[GET/POST/PUT/DELETE]
路径：[/api/xxx]
请求参数：
  - 参数名 | 类型 | 是否必填 | 说明 | 约束
  [逐行列出]
响应结构：
  {"code": 0, "data": {...}, "message": ""}

生成要求同方式一，额外要求：
- 结合 test_data/baseline.sql 中的基准数据设计参数值
- 对于"创建/修改"类接口，追加数据库断言（查询 MySQL 验证实际写入）
```
