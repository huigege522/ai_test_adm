# 一、这个系统到底是干什么的？

这是一个：

# 苹果 ADM 投放基础数据管理平台

主要管理：

```text id="nl4e0d"
客户
↓
广告账户（广告系列组）
↓
APP产品
↓
账户与产品关系
↓
新客状态
```

核心目标：

* 统一账户管理
* 统一产品管理
* 支持OAuth账户
* 支持代理体系
* 支持产品分类
* 支持新客状态管理
* 支持运营批量维护

---

# 二、整体业务模型（非常重要）

# 1）客户与账户

```text id="q6ng7f"
客户 1 : N 账户
```

一个客户：

* 可以有多个广告账户

---

# 2）账户与产品

```text id="j7s0vw"
账户 N : N 产品
```

一个账户：

* 可以投多个APP

一个APP：

* 也可以挂多个账户

---

# 3）关系表

系统存在：

```text id="vngvgv"
账户 + 产品 + 客户属性
```

关系管理表。

---

# 三、系统模块

系统分成：

| 模块       | 作用        |
| -------- | --------- |
| 账户管理     | 管广告账户     |

# 十、新客标识逻辑（重点）

---

# 状态类型

| 状态     | 含义       |
| ------ | -------- |
| 全新客    | 真正新客户    |
| 回流客    | 老客户回归    |
| 老客     | 长期稳定投放   |
| 新客孵化中  | 有消耗但未达标  |
| 未分配/失效 | 无消耗/长期流失 |

---

# 状态流转

---

## 1）默认状态

新APP默认：

```text id="u63av0"
未分配/失效
```

---

## 2）孵化中

条件：

```text id="iqnlk0"
有消耗
但未达新客条件
```

---

## 3）全新客（重点）

现在：

# 只能手动设置

条件：

```text id="z47gmc"
0 < 过去2季度消耗 < 1000美元
```

系统：

开启90天返点。

---

## 4）老客

条件：

```text id="0dwoc0"
Net New 满90天
且连续2季度 >1000美元
```

系统自动转老客。

---

## 5）回流客

条件：

```text id="kr6x9v"
历史老客
上季度消耗=0
本季度重新有消耗
```

---

## 6）失效

条件：

```text id="5r8ny2"
180天无消耗
```

变：

```text id="g0nh1m"
Inactive
```
