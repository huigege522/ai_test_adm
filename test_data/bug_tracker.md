# 缺陷台账（Bug Tracker）

> 与 [prompts/07_defect_analysis.md](../prompts/07_defect_analysis.md) **阶段六**配合使用：记录 pytest / E2E 失败后经 AI 辅助分析的结论。建议在「团队缺陷系统」中建单后，在此填写同一编号以便追溯。

---

## 字段说明（与阶段六模板一致）

| 字段 | 说明 |
|------|------|
| 发现时间 | `YYYY-MM-DD` |
| 失败用例 | pytest node id，例如 `tests/api/test_xxx.py::TestClass::test_yyy`；手工发现可写 `手工` + 一句复现 |
| AI 分析根因 | 摘录阶段六 Prompt A 输出中的根因判断 |
| 缺陷定位 | 代码路径与位置，例如 `SomeService.java:L120` 或 `包.类#方法` |
| 修复状态 | `待修复` / `已修复(PR#xxx)` / `转需求` / `已关闭`（按需扩展） |

**可选**（便于和测试点、对外系统对齐）：

| 字段 | 说明 |
|------|------|
| 团队缺陷单 | 如 `JIRA-123`、`禅道-456`，无则填 `无` |
| 关联测试点 | 如 `TC001`（见 `test_data/testpoints/{模块}/testpoints_*.md`） |
